# Designer corpus

Role-specific knowledge corpus for the Designer (Product Designer) agent. Designer owns Notion design specs and `design-prototype/` directories — all design work is code in the app's actual stack (Tailwind + React for richezamor.com, HTMX + Jinja + Pico CSS for SIA), not Figma. UI Eng productionizes prototypes.

## Index

- **[don-norman.md](./don-norman.md)** — Foundational interaction design: affordances, signifiers, feedback, mapping, constraints, error prevention. The vocabulary behind every interactive element the Designer writes.
- **[luke-wroblewski.md](./luke-wroblewski.md)** — Two decades of practical work on forms, mobile-first, and input patterns. Directly executable guidance for the Designer's highest-value work (forms, sign-up, settings).
- **[brad-frost.md](./brad-frost.md)** — Atomic Design — atoms/molecules/organisms/templates/pages. Maps one-to-one onto prototype file structure in both React and Jinja.
- **[adam-wathan-tailwind.md](./adam-wathan-tailwind.md)** — Utility-first CSS and Tailwind as the richezamor.com design system. Paired with *Refactoring UI* (Wathan + Schoger) for applied visual craft.
- **[josh-comeau.md](./josh-comeau.md)** — Modern CSS, layout, animation, and interaction craft. The "why it actually works" layer under Tailwind utilities.
- **[heydon-pickering-adrian-roselli.md](./heydon-pickering-adrian-roselli.md)** — Authoritative accessible interaction patterns with tested, working code. Non-negotiable for prototypes UI Eng will ship.
- **[smashing-alistapart.md](./smashing-alistapart.md)** — Two long-running publications for design/dev craft — responsive design, progressive enhancement, design systems, UX decision trees.
