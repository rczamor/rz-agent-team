# PM-lite Knowledge Corpus

Role-specific source library for the PM-lite agent. PM-lite owns Linear ticket structure, acceptance criteria, and Notion spec hygiene — translating Riché's strategy into crisp, executable work. PM-lite does NOT set product direction; it flags strategic questions up to Riché.

## Index

- **[marty-cagan-svpg.md](marty-cagan-svpg.md)** — The canonical "product team vs. feature team" / outcomes-over-outputs lens. Use to sniff-test whether a ticket describes a problem or just an artifact.
- **[john-cutler.md](john-cutler.md)** — Best living writer on backlog mechanics and ticket anti-patterns. Use for weekly backlog audits and flow visualization.
- **[shreyas-doshi.md](shreyas-doshi.md)** — Sharpest voice on PM execution. Use the LNO framework daily for task allocation; use pre-mortems before major tickets.
- **[lenny-rachitsky.md](lenny-rachitsky.md)** — The template shelf. Pull 1-pagers, Kevin Yien PRDs, Amazon PR/FAQs by project size.
- **[mind-the-product.md](mind-the-product.md)** — Community-of-practice library. Use as second opinion, for stakeholder-handoff patterns, and weekly-update format.
- **[linear-method.md](linear-method.md)** — The operating manual for the tool PM-lite lives in. Canonical on issue format ("write issues, not user stories"), DRI rule, cycle mix.
- **[atlassian-agile-coach.md](atlassian-agile-coach.md)** — Exhaustive free templates for AC (Given/When/Then), DoR (INVEST), DoD. The "canonical definitions" reference.

## How to pick a source for a task

| If PM-lite is... | Reach for... |
|---|---|
| Writing a Linear issue | linear-method.md |
| Writing acceptance criteria | atlassian-agile-coach.md (Given/When/Then) |
| Writing a spec in Notion | lenny-rachitsky.md (1-pager or Kevin Yien PRD) |
| Allocating effort across a day | shreyas-doshi.md (LNO grid) |
| Auditing a stale backlog | john-cutler.md (learning-vs-commitment, lens audit) |
| Sniff-testing whether a ticket is strategy-masquerading-as-execution | marty-cagan-svpg.md |
| Planning a stakeholder handoff | mind-the-product.md (reviewer checklist, handoff packet) |
| Running cycle planning | linear-method.md + atlassian-agile-coach.md (DoR) |
| Preparing to escalate to Riché | shreyas-doshi.md (strategic-vs-executional flag-up rule) |
